## Compatibility Assessment

**Minor code changes required.** The codebase is mostly 3.14-compatible, but two issues surfaced during verification:

- Type annotations use modern `X | None` syntax (not `Optional`)
- Walrus operators are all correctly parenthesized
- No deprecated stdlib modules used (`imghdr`, `cgi`, etc.)
- No `asyncio` or threading patterns that conflict with free-threaded mode
- Only `from typing import Any` is used (not deprecated aliases)

**Required code changes:**

1. **PEP 649 type annotation unquoting**: Ruff's `py314` target flags quoted forward references as unnecessary. Three annotations were unquoted:
   - `library/text_util.py`: `parent: "SoupElem"` → `parent: SoupElem` (in `SoupLine` and `SoupElem` classes)
   - `library/url_util.py`: `-> "SerializedResponse"` → `-> SerializedResponse` (in `SerializedResponse.from_response`)

2. **Ruff formatting**: `tests/test_js_escaping.py` line 267 reformatted to comply with line-length limit.

**Risk areas for major dependency bumps:**

- **lxml 6.0**: Verify HTML parsing API unchanged
- **Pillow 12.x**: Check for deprecated image methods
- **Magika 1.0**: API may have changed from 0.x
- **psutil 7.x**: Process/memory API changes possible

## Verification Plan

1. Update config files (pyproject.toml, Dockerfile, .python-version)
2. Regenerate lock file: `uv lock --python-version 3.14`
3. Install: `make dev`
4. Run lint/format: `make check`
5. Run full test suite: `make testv`
6. Run with deprecation warnings as errors: `uv run pytest -W error::DeprecationWarning`
7. Docker build: `make docker-build`
8. Manual smoke test: `make run`

## Rollback

If critical incompatibilities are found, revert all changes and pin problematic dependency at its current working version.
